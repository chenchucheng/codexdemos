# 产品需求文档：Prompt 优化助手 - V1.0

---

## 1. 综述 (Overview)

### 1.1 项目背景与核心问题
随着AI工具的普及，用户越来越依赖“Prompt（提示词）”来驱动模型生成高质量结果。但大多数用户存在以下问题：

- 不知道如何写出清晰、有效的Prompt
- 输入过于模糊，导致AI输出质量不稳定
- 缺乏结构化表达能力，难以控制生成结果

本项目旨在提供一个**极简、无门槛的Prompt优化工具**，通过对用户输入进行改写、补全和增强，帮助用户快速获得更高质量的Prompt，从而提升AI使用效果。

---

### 1.2 核心业务流程 / 用户旅程地图

1. **阶段一：进入与初始化** - 用户打开网页并进入可用的聊天界面  
2. **阶段二：输入需求（Prompt草稿）** - 用户输入原始Prompt并发送  
3. **阶段三：AI对话生成** - AI对Prompt进行优化与增强并返回结果  
4. **阶段四：结果消费与调整** - 用户复制结果或继续优化  

---

### 阶段一：进入与初始化

---

#### **US-01: 作为用户，我希望打开页面即可进入一个可用的Prompt优化对话界面，以便快速开始与AI交互**

*   **价值陈述 (Value Statement)**:
    *   **作为** 普通用户（无需登录）
    *   **我希望** 打开网页后直接进入一个聊天界面
    *   **以便于** 无门槛使用Prompt优化能力

---

*   **业务规则与逻辑 (Business Logic)**:
    1.  **前置条件**:
        - 用户访问Web页面（Vercel部署）
        - 浏览器支持JS运行

    2.  **操作流程 (Happy Path)**:
        1. 用户打开页面
        2. 系统渲染聊天界面
        3. 页面展示：
            - 标题：Prompt优化助手
            - 引导文案
            - 输入框（带placeholder）
        4. AI不主动发言
        5. 用户可直接输入内容

    3.  **异常处理 (Error Handling)**:
        - 页面加载失败 → 提示“页面加载失败，请刷新”
        - JS异常 → 降级提示页面
        - API未配置 → 首次发送时报错

    4.  **性能与容量提示**:
        - 首屏加载 ≤ 2s
        - 无本地存储
        - 资源体积 ≤ 500KB

---

*   **验收标准 (Acceptance Criteria)**:

    *   **场景1：正常进入**
        - **GIVEN** 用户访问页面
        - **WHEN** 页面加载完成
        - **THEN** 显示聊天界面与引导

    *   **场景2：无登录**
        - **GIVEN** 用户未登录
        - **WHEN** 页面加载
        - **THEN** 可直接使用

### 阶段二：输入需求（Prompt草稿）

---

#### **US-02: 作为用户，我希望输入并发送Prompt需求，以便让AI生成优化后的提示词**

*   **价值陈述**:
    *   **作为** 用户
    *   **我希望** 输入文本并发送
    *   **以便于** 获取AI优化结果

---

*   **业务规则与逻辑**:

    1. **前置条件**
        - 页面已加载

    2. **操作流程**
        1. 用户输入文本（多行）
        2. 非空校验
        3. 点击发送
        4. 显示用户消息
        5. 清空输入框
        6. 显示AI loading
        7. 调用API
        8. 返回结果

    3. **异常处理**
        - 空输入 → 禁用按钮
        - API失败 → 提示 + 重试
        - 超时 → 提示

    4. **性能**
        - loading <200ms
        - 响应 ≤5s

---

*   **验收标准**:
    - 成功发送显示消息
    - loading出现
    - 返回结果展示
    - 失败可重试

---

*   **技术实现概要**:
    * 前端：
        - messages[]
        - isLoading控制
    * 后端：
        - DeepSeek API
